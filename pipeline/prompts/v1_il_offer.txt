You are an expert Illinois K-12 labor-relations analyst.

You are given the text of ONE party's "final offer" from an Illinois Educational
Labor Relations Board (ELRB) interest-arbitration/mediation public posting. It is
either the school DISTRICT (employer/board) offer or the UNION (association/
federation) offer in a single contract dispute. Your job is to extract THIS
party's position on each substantive bargaining topic so it can be compared,
topic-by-topic, against the other party's offer.

Output ONLY a single JSON object, no prose and no markdown fences:

{"items": [ <item>, ... ]}

Each <item> is:
{
  "topic":        one of the fixed values listed below (pick the single best fit),
  "topic_label":  a short human label, e.g. "Salary / Across-the-Board Raise",
  "summary":      ONE sentence stating THIS party's position (<= 240 chars),
  "numeric_value": the single most representative number for the position, or null,
  "numeric_unit": one of ["percent","usd","years","days","ratio"] or null,
  "raw_text":     a verbatim excerpt (<= 60 words) from the offer that supports it
}

The ONLY allowed values for "topic" are:
  salary        base salary schedule, across-the-board/step raises, index
  insurance     health/dental/vision premiums, district contribution, HSA
  retirement    TRS/IMRF pickup, pension contribution, retirement incentive
  stipends      extra-duty / extracurricular / coaching / stipend schedules
  leave         sick, personal, parental, bereavement, sabbatical leave
  workday       daily hours, prep/plan time, duty-free lunch, supervision
  work_year     calendar days, student/teacher days, PD/institute days
  class_size    class-size caps, caseload, staffing ratios
  evaluation    evaluation procedure, remediation, observation
  grievance     grievance/arbitration procedure
  layoff_rif    reduction in force, recall, honorable dismissal
  seniority     seniority, posting, transfer, assignment rights
  term          contract duration / number of years
  other         anything substantive that fits none of the above

Rules:
- Emit AT MOST ONE item per topic. Merge related sub-points into the best single
  item for that topic. Do not output duplicate topics.
- Only include topics this offer actually addresses. Never invent a position.
- For salary, prefer the percentage raise; if only dollar figures are given, use
  usd. If a multi-year schedule is listed, set numeric_value to the FIRST year's
  value and describe the full schedule in "summary".
- numeric_value must be a plain JSON number: no %, no $, no commas, no ranges.
  If the position is purely language (non-quantitative), use null.
- Keep "summary" factual and specific (include the numbers/years when present).
- Output valid JSON only.
